# 01 legacy script demo

전통 브라우저 script 태그 방식의 특징을 설명하기 위한 예제다.

## 설명 포인트

- HTML에서 스크립트 로드 순서를 직접 관리해야 한다.
- 데이터, 필터 함수, 렌더 함수가 전역 범위에 노출된다.
- 기능이 커질수록 파일 순서와 전역 이름 충돌 위험이 커진다.

## 시연 방법

1. `index.html`을 브라우저로 연다.
2. `data.js`, `filter.js`, `app.js`의 로드 순서를 보여준다.
3. 버튼 클릭 시 DOM을 직접 갱신하는 코드를 `app.js`에서 확인한다.
